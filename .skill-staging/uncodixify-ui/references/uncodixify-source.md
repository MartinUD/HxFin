## Intent

Use this skill to avoid the visual defaults that make interfaces look machine-generated: soft gradients, floating shells, decorative dashboard filler, oversized radii, ornamental labels, and forced "premium SaaS" styling.

## Keep It Normal

- Sidebars: 240-260px fixed width, solid background, simple border-right, no detached shell
- Headers: plain hierarchy, no eyebrow labels, uppercase labels, or gradient text
- Sections: standard padding, no hero block inside product UI
- Navigation: simple links, subtle hover, no transform animations or decorative badges
- Buttons: solid or outlined, 8-10px radius max, no pill shapes, no gradients
- Cards: subtle border, 8-12px radius max, restrained shadow only
- Forms and inputs: visible labels, solid borders, clear focus state, no floating labels
- Modals and dropdowns: straightforward overlays and lists, minimal animation
- Tables and lists: left-aligned, clean rows, clear hierarchy, no unnecessary tag clutter
- Tabs and badges: simple indicators, no pill treatments or glow
- Typography: readable sans or product-appropriate type, no mixed "premium" shortcut stacks
- Spacing and layout: consistent scale, predictable grid/flex structure, no decorative wrappers
- Panels and toolbars: functional only, no glass, no floating detachment

## Hard No

- Oversized rounded corners
- Pill overload
- Glassmorphism by default
- Soft corporate gradients used as taste-substitute
- Generic dark SaaS composition
- Decorative blobs, glows, haze, or frosted surfaces
- Control-room dashboard cosplay
- Serif headline plus safe sans fallback as fake premium
- Default font stacks unless the product already uses them
- Sticky left rails without information-architecture need
- Metric-card grids as the first instinct
- Fake charts that only fill space
- Hero sections inside internal UI
- Random ornamental labels or startup copy
- Styling choices made only because they are easy to generate
- `small` headings and rounded `span` tags used decoratively

## Specifically Banned Patterns

- 20-32px radii applied broadly across the interface
- Same rounded rectangle language on sidebar, cards, buttons, and panels
- Detached rounded sidebar shells with brand block on top
- Glass cards around charts without a product reason
- Donut charts with vague percentages
- Glows instead of hierarchy
- Mixed alignment logic that produces dead space
- Muted gray-blue text that weakens contrast
- Blue-black gradients plus cyan accents sold as "premium dark mode"
- Eyebrow labels, uppercase labels, and letter-spacing-heavy metadata
- Decorative copy explaining what each section does
- Hover transforms on nav links
- Dramatic shadows
- Pseudo-element status dots
- Gradient progress bars and generic KPI cards
- Over-tagged tables
- Nav badges, live markers, and usage/quota fillers
- Right rails for "today" schedules unless required
- Too many panel variants in one screen

## Color Guidance

- Keep colors calm.
- Prefer muted dark or light palettes with clear text contrast.
- Avoid default blue-forward SaaS color systems unless the existing product already uses one.
- Use accent colors sparingly and functionally.

## Palette Hints From The Source

The source includes several dark and light palette suggestions. Treat them as starting points, not prescriptions. Choose one only if it suits the product and still respects the rest of the rules.
